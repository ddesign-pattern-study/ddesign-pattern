< 싱글톤 >

- 멀티 스레드 환경에서의 싱글톤 구현은 주의해야할 부분이 있다.
1. 여러 스레드 간의 공유
    - 각 스레드 별로 성능 향상을 위해 객체를 cpu cache 에 저장하기 때문에 스레드 간의 공유가 되지 않는다.
    - 때문에 스레드 간의 공유를 위해 main memory 에 저장하고 사용하기 위해서 volatile 키워드를 선언야한다.
    - volatile 키워드를 통한 해결방법은 멀티 스레드 환경에서 하나의 스레드만 write 하고 나머지 스레드가 read 하는 상황에서 적합하다.
    - 여러 스레드가 쓰기를 할 경우에는 volatile 만으로는 불가하고 모니터 락을 걸어 원자성을 보장해야한다.
